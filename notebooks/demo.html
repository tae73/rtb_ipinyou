▶ RTB win-selection-bias debiasing → bidding — a runnable demo¶

What this shows, end-to-end, in ~1 minute (CPU only, no 25G data, no GPU):

  1. Real usage — a live RTB micro-pipeline: a market-calibrated testbed → a biased vs debiased click model → a bid (bid = p̂·CPC) → an auction → business metrics (surplus, win-rate, unprofitable-win share, ROI). The same decision is also run through the production auction simulator.
  2. The recalibration trap — naively recalibrating the biased model over-bids and loses money.
  3. Bid shading — a practical lever, with its honest caveat.
  4. The findings — the phase diagram (C1) and the neural anchor (real ESCM²-WC: the censoring-bug fix and the calibration 2×2), loaded verbatim from the committed witness JSONs.

Why a semi-synthetic testbed? Decision value (did debiasing change the bid for the better?) is unmeasurable on real iPinYou — flat-bid logging censors the lost inventory. So we use real iPinYou market statistics + a controllable ground-truth pCTR with observable lost inventory. Every number here is reproducible from requirements.txt; neural results load from JSON (see repro/check.py).

1. The market and the funnel¶

A bid passes Bid → Win → Click: you only observe a click on impressions you won. So a click model trained on winners is selection-biased, and biased pCTR → biased bids. The testbed's market price is a lognormal calibrated to the real iPinYou price distribution below (median ≈ 68 CPM).

2. A live RTB micro-pipeline (real usage)¶

Generate one market with ground-truth pCTR and observable lost inventory, fit a biased (winners-only) and an IPW-debiased click model at the same capacity, bid truthfully (bid = p̂·CPC), and read off the decision metrics vs the oracle (true pCTR). Strong selection (γ=1.2), linear model.

In [3]:
def make_world(gamma=1.2, theta=0, seed=0):
    """Real-market-calibrated testbed: ground-truth pCTR + observable lost inventory + a win-propensity model."""
    X, pctr, click, mprice = P.make_pop(gamma, theta, seed)
    b0 = np.quantile(mprice, 0.5)
    win = (b0 >= mprice).astype(int)                 # which bids won (flat bid at the market median)
    pwin = LGBMClassifier(n_estimators=150, num_leaves=31, min_child_samples=80,
                          verbose=-1).fit(X, win).predict_proba(X)[:, 1]   # P(win | x)
    return X, pctr, click, mprice, win, pwin

X, pctr, click, mprice, win, pwin = make_world(gamma=1.2, seed=0)
CAP = "linear"
p_oracle = pctr
p_biased = P.fit_pred(X, win, click, CAP, False, pwin)   # winners-only → selection-biased
p_debias = P.fit_pred(X, win, click, CAP, True,  pwin)   # winners + IPW(1/P(win)) → debiased

def decision_metrics(name, phat):
    d = R.decompose(phat, pctr, mprice)                  # truthful bid = phat·CPC, 2nd-price surplus
    return {"model": name, "mean_bid": d["mean_bid"], "win_rate": d["n_wins"] / len(pctr),
            "unprofitable_win_share": d["unprofitable_win_share"], "won_surplus_$M": d["won_surplus"] / 1e6}

print(f"win rate {win.mean():.2f} | true pCTR mean {pctr.mean():.3f} | "
      f"biased pCTR mean {p_biased.mean():.3f} (collapsed by selection) | debiased mean {p_debias.mean():.3f}")
pd.DataFrame([decision_metrics("oracle (true pCTR)", p_oracle),

def production_sim(name, phat):
    bids, values = phat * CPC, pctr * CPC
    res = run_auction_simulation(bids=bids, market_prices=mprice, values=values, clicks=click,
                                 auction_type="second_price")
    mt = compute_simulation_metrics(res, values=values, market_prices=mprice, strategy_name=name, cpc_target=CPC)
    return {"strategy": name, "win_rate": mt.win_rate, "total_surplus_$M": mt.total_surplus / 1e6,
            "overpayment_ratio": mt.overpayment_ratio, "ROI": mt.roi}

4. Bid shading (a practical lever)¶

Real bidders rarely bid p̂·CPC verbatim — they apply linear shading bid = α·p̂·CPC. Best-case shading finds the α that maximizes surplus. Honest caveat: optimal shading rewards spread, so report it alongside truthful bidding, never instead (it can flatter a miscalibrated model — see §5).

In [6]:
ALPHAS = np.concatenate([np.linspace(0.05, 1.0, 20), np.linspace(1.1, 4.0, 30)])
best_shaded = lambda phat: max(P.surplus(phat, pctr, mprice, alpha=a) for a in ALPHAS)
s_or = P.surplus(pctr, pctr, mprice)
for name, phat in [("biased", p_biased), ("debiased (IPW)", p_debias)]:
    t, sh = P.surplus(phat, pctr, mprice), best_shaded(phat)
    print(f"{name:16}  truthful {t/1e6:5.2f}M   best-shaded {sh/1e6:5.2f}M   (oracle {s_or/1e6:.2f}M)")

5. The findings — neural anchor (real iPinYou features + the real ESCM²-WC)¶

Loaded verbatim from witnesses/neural_anchor.json (the heavy GPU training is not re-run here). Two honest, hard-won results — each a self-correction caught by adversarial review:

  • The "over-bidding" was a data-contract bug. A first pass showed the neural debiaser over-bidding (truthful edge −47pp). It was mostly a censoring wiring bug (the loss expects click censored to 0 on losses). Fixed (click·win) → the real ESCM²-WC genuinely helps truthful bidding (+7.2pp).
  • A calibration 2×2 — does selection-aware IPW calibration beat naive? Only at weak selection (good overlap); it loses at strong selection (low ESS). Net, naive ties it. **Lesson: calibration accuracy ≠ bidding *value*** (a level-correct calibration over-bids near the margin).

The mechanism, live (and why it's regime-dependent): both calibrators shift the level; which lands closer to truth depends on the regime (overlap/ESS, model). On the real-feature neural 2×2 above, IPW lifts more than naive; on this Gaussian cell it can be the reverse. The robust point isn't "which calibrator is more accurate" — it's that a level shift drives the bid, so a level-accurate calibration is not reliably a better bidding one near the 2nd-price margin (accuracy ≠ bidding value).

def calibrate_ipw(p, click, win, pwin):
    w = win == 1
    sw = 1.0 / np.clip(pwin[w], 0.05, 1.0); sw = sw / sw.mean()
    return IsotonicRegression(out_of_bounds="clip").fit(p[w], click[w], sample_weight=sw).transform(p)

p_naive = P.xfit_isotonic(p_biased[win == 1], click[win == 1], p_biased)
p_ipw   = calibrate_ipw(p_biased, click, win, pwin)
print(f"biased pCTR mean {p_biased.mean():.3f}  |  +naive-cal {p_naive.mean():.3f}  |  "
      f"+IPW-cal {p_ipw.mean():.3f}  |  true {pctr.mean():.3f}")

7. Takeaways¶

  • Real usage: win-selection bias collapses a winners-only click model → it under-bids; IPW debiasing recovers it and wins more profitable inventory (live, above). The production auction simulator agrees.
  • Recalibration trap: naively recalibrating a biased model over-bids into unprofitable inventory and lowers surplus — a real, robust failure mode of the obvious fix.
  • Calibration accuracy ≠ bidding value: even the principled selection-aware (IPW) calibration only pays when overlap is good; near the 2nd-price margin a level-correct calibration over-bids. For truthful bidding, conservative under-calibration is safer.
  • On real iPinYou features + the real ESCM²-WC: correctly wired, debiasing genuinely helps truthful bidding (+7.2pp) — the earlier "−47pp" was a data-contract bug, not the method.

Honest scope. Semi-synthetic (real market + features, synthesized selection; ground-truth pCTR is a fitted surrogate); decision value is unmeasurable on real iPinYou (the data ceiling); neural results use n=2 seeds (trust the sign/direction, not the magnitude).
